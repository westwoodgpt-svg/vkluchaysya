# «Включайся» — память проекта

Приложение для сбора идей сотрудников: локальное приложение Битрикс24 (iframe),
которое создаёт лид «Идея № N — …» и раскладывает ответы формы по полям CRM.

- **Репозиторий:** https://github.com/westwoodgpt-svg/vkluchaysya
- **Деплой:** https://vkluchaysya.vercel.app (auto-deploy при push в `main`)
- **Портал:** portal39.mb-product.ru (коробочный Битрикс24), приложение — `/marketplace/app/73/`
- **Стек:** React 19 + Vite + Tailwind (SPA) + Vercel serverless (`api/`)

## Архитектура

- `api/index.ts` — обработчик корня `/`. Отдаёт собранную SPA и на GET, и на POST
  (Битрикс24 загружает iframe POST-запросом с AUTH_ID и добавляет к URL свой
  `APP_SID`). Auth из POST внедряется в страницу как `window.__BX_AUTH__`.
  **Редиректа быть не должно** — он теряет `APP_SID`, без которого не работает
  handshake JS SDK (`BX24.*`).
- `api/rest.ts` — прокси REST-вызовов (обход CORS), запасной путь, когда SDK недоступен.
- `vercel.json` — rewrite `/` → `/api`; сборка переименовывает `dist/index.html`
  → `dist/app.html`, чтобы статика не перехватывала корень.
- `src/utils/bx24.ts` — `whenBx24Ready()` (ожидание handshake + `BX24.installFinish()`),
  `getAuth()` с приоритетом: `BX24.getAuth()` → `__BX_AUTH__` → URL-параметры → localStorage.
- `src/App.tsx` — `ensureCrmFields()`: при загрузке и **перед каждой отправкой**
  находит/создаёт поля лида и строит маппинг. Меню настройки CRM удалено —
  всё автоматически.

## Поля CRM (лиды)

8 строковых UF-полей, созданы 2026-07-14 автоматически приложением:

| Код | Название |
|---|---|
| UF_CRM_IDEA_DEPT | Включайся: Отдел |
| UF_CRM_IDEA_CAT | Включайся: Категория идеи |
| UF_CRM_IDEA_SIT | Включайся: Текущая ситуация |
| UF_CRM_IDEA_DESC | Включайся: Суть идеи |
| UF_CRM_IDEA_LINK | Включайся: Ссылка на материалы |
| UF_CRM_IDEA_FIN | Включайся: Финансовые затраты |
| UF_CRM_IDEA_IMPL | Включайся: Готовность внедрить |
| UF_CRM_IDEA_AUD | Включайся: Целевая аудитория |

Маппинг ищет поле по коду **или** по точному названию «Включайся: …».
ФИО/должность пишутся в штатные поля лида (LAST_NAME/NAME/SECOND_NAME/POST).
Полный текст идеи всегда дублируется в COMMENTS как страховка.
`ASSIGNED_BY_ID = 1220` захардкожен в `App.tsx` (ответственный за лиды).

## Грабли (не наступать повторно)

1. **Не возвращать редирект в `api/index.ts`** — сломается BX24 SDK (`APP_SID`).
2. Создание UF-полей требует прав **администратора CRM**; у рядовых сотрудников
   попытка молча падает, и данные идут только в COMMENTS (плашка на шаге 3 формы).
3. Заполненные UF-поля **не видны в карточке лида**, пока их не добавили в
   раскладку: карточка лида → «Выбрать поле» → отметить поля «Включайся: …»
   (и сохранить вид «для всех»).
4. Поля созданы только для **лидов** — при конвертации в сделку не переносятся
   (нет парных полей в сделках).
5. QR-код (`Downloads\qr-vkluchaysya.png` → `/marketplace/app/73/`) открывает
   браузер с формой входа: коробочный домен не перехватывается мобильным
   приложением Битрикс24, deep-link невозможен. Вход в браузере нужен один раз.
